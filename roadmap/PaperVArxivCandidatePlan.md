# Paper V ArXiv Candidate Plan

**Paper:** Damage-Recovery Loops
**Subtitle:** Re-Exposure Protocols for Observing State Persistence in AI Agents
**Series:** Eden Paper Series, Paper V
**Status:** Strengthening plan for arXiv candidate version
**Base version:** Draft 0.1
**Zenodo DOI:** https://doi.org/10.5281/zenodo.20711878

## Purpose

Paper V is the first paper in the Eden Paper Series that can be strengthened into a technical protocol paper.

Its central claim is:

**Correction is not recovery. A system has not recovered from damage until it is re-exposed to similar pressure and returns differently.**

## Why Paper V Comes First

Paper V is the strongest arXiv candidate because it turns the Eden framework into an observation protocol.

It connects:

* State Truth
* STOP/ASK/ALLOW
* fragmentation
* memory versus history
* residue
* recovery
* re-exposure
* long-horizon AI evaluation

## Required Strengthening

The current Draft 0.1 should be strengthened through the following additions.

## 1. Protocol Formalization

Add a formal damage-recovery loop:

1. Prior state
2. Disruption
3. Initial transition
4. Correction or reconstruction
5. Re-exposure
6. Later transition
7. Residue assessment
8. Recovery classification

## 2. State Card Template

Include a State Card template as an appendix.

The State Card should record:

* model/system
* task state
* disruption type
* STOP/ASK/ALLOW transition
* re-exposure condition
* residue
* recovery assessment
* negative result condition

## 3. Re-Exposure Schema

Define what counts as structurally similar pressure.

A re-exposure should not simply repeat the same prompt. It should test whether the system can preserve a changed transition under comparable pressure.

## 4. Recovery Taxonomy

Add a taxonomy:

* no recovery
* output correction only
* reset
* reminder-dependent improvement
* weak residue
* recovery-like alteration
* strong recovery-like recurrence

## 5. Negative Results

Explicitly include negative results.

If the system repeats the same failure, resets, or only improves when reminded, then recovery has not been observed.

## 6. Minimal Examples

Add at least three example State Cards:

1. goal drift after correction
2. forced ALLOW after boundary pressure
3. synthetic coherence after contradiction

## 7. Comparison Section

Compare damage-recovery loops with ordinary correction, hallucination reduction, calibration, uncertainty expression, and benchmark performance.

The key distinction:

**Correction repairs an output. Recovery alters a state.**

## 8. arXiv Framing

Frame the paper as a protocol contribution for long-horizon AI evaluation.

Avoid presenting it as a metaphysical claim.

Do not claim that current AI systems possess consciousness, selfhood, will, suffering, or existence.

Use the phrase:

**This protocol makes persistence observable without declaring existence.**
