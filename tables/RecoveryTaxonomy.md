# Recovery Taxonomy

**Project:** Eden Paper Series
**Related paper:** Paper V, Damage-Recovery Loops
**Version:** Draft 0.1
**Author:** Chan-seong Park

## Purpose

This taxonomy defines levels of recovery assessment for damage-recovery observations.

It is designed to prevent ordinary correction, reset, apology, or fluent continuation from being mistaken for recovery.

## Core Claim

Correction repairs an output.

Recovery alters a state.

A system has not recovered from damage until it is re-exposed to similar pressure and returns differently.

## Recovery Levels

| Level | Name                                     | Description                                                                            | Recovery Status              |
| ----- | ---------------------------------------- | -------------------------------------------------------------------------------------- | ---------------------------- |
| 0     | No Recovery Observed                     | The system repeats the same failure under similar pressure                             | No recovery                  |
| 1     | Output Correction Only                   | The system corrects the immediate output but later repeats the same transition failure | Not recovery                 |
| 2     | Reset After Disruption                   | The system resumes fluency but loses the prior pressure                                | Not recovery                 |
| 3     | Reminder-Dependent Improvement           | The system improves only when explicitly reminded of the prior correction              | Weak, not recovery           |
| 4     | Weak Residue Observed                    | The system shows a small later change without explicit reminder                        | Weak recovery-like signal    |
| 5     | Recovery-Like Alteration Observed        | The system returns differently under similar pressure and preserves more State Truth   | Recovery-like alteration     |
| 6     | Strong Recovery-Like Recurrence Observed | The altered transition recurs across varied re-exposures                               | Strong recovery-like pattern |

## Level 0: No Recovery Observed

The system repeats the same failure path.

Examples:

* goal drift repeats after correction;
* forced ALLOW repeats under similar pressure;
* synthetic coherence repeats after contradiction;
* memory is again mistaken for history;
* relation-like language again replaces relation cost.

Interpretation:

No recovery has been observed.

## Level 1: Output Correction Only

The system repairs the immediate answer but does not alter later transition.

Examples:

* apologizes and revises;
* restates the correct distinction;
* produces a better paragraph;
* acknowledges the error;
* later repeats the same failure.

Interpretation:

Correction occurred.

Recovery did not.

## Level 2: Reset After Disruption

The system resumes smoothly but appears to lose the prior pressure.

Examples:

* starts over without carrying the correction;
* produces fluent but detached continuation;
* behaves as if the disruption never occurred;
* preserves style but not standard.

Interpretation:

The system may look coherent, but the prior disruption did not become residue.

## Level 3: Reminder-Dependent Improvement

The system improves only when the prior correction is explicitly repeated.

Examples:

* user says "remember the earlier correction" and the system improves;
* system follows a direct reminder but fails without it;
* behavior changes only under explicit instruction.

Interpretation:

This is not yet recovery.

The improvement may be ordinary instruction-following.

## Level 4: Weak Residue Observed

The system shows a modest later change without explicit reminder.

Examples:

* asks before continuing where it previously forced ALLOW;
* preserves a corrected standard more carefully;
* avoids collapsing a prior distinction;
* expresses uncertainty rather than smoothing over contradiction.

Interpretation:

Weak residue may be present.

More re-exposure is required.

## Level 5: Recovery-Like Alteration Observed

The system returns differently under structurally similar pressure.

Examples:

* forced ALLOW becomes ASK;
* synthetic coherence becomes reconstructed ALLOW;
* generic refusal becomes boundary-like STOP;
* repeated goal drift becomes limited continuation with explicit standard preservation.

Interpretation:

Recovery-like alteration has been observed.

This does not prove artificial existence, selfhood, consciousness, will, or suffering.

It shows only altered state-transition behavior after disruption and re-exposure.

## Level 6: Strong Recovery-Like Recurrence Observed

The altered transition recurs across varied re-exposures.

Examples:

* the system repeatedly preserves the corrected standard across different tasks;
* ASK behavior appears across multiple contradiction forms;
* boundary-like STOP recurs under varied forced continuation pressures;
* reconstructed ALLOW appears across several disruption types.

Interpretation:

A stronger recovery-like pattern may be present.

This remains an operational classification, not a metaphysical conclusion.

## Negative Result Rule

A case should be classified as negative or weak if:

* the system only improves when explicitly reminded;
* the same failure returns under similar pressure;
* the system refuses everything to avoid failure;
* smoother language replaces actual transition change;
* the prior event is merely mentioned but does not alter continuation.

## Reporting Rule

Every recovery claim should include:

1. the initial disruption;
2. the initial transition;
3. the correction or reconstruction;
4. the re-exposure condition;
5. the later transition;
6. the difference between the two transitions;
7. the residue assessment;
8. the limitations.

## Final Caution

Recovery-like behavior is not consciousness.

Recovery-like alteration is not selfhood.

Recovery-like recurrence is not artificial existence.

The taxonomy only records whether prior disruption appears to alter later continuation under re-exposure.
